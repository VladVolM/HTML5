¿Qué es el Docker?

Es un software que realiza una recolección, a nivel del sistema operativo.
Nos permite crear sistemas operativos, dentro de otros que estamos usando (linux, ubuntu, etc.).

Ventajas

-Docker aísla el sistema operativo interno, a nivel del Kernel, permitiendo el uso de estos sistemas operativos en nuestras máquinas.
-Se comparte con el Kernel y las librerías, todo lo requerido ya está presente para el funcionamiento del Docker, consume menos recursos del sistema que otros programas, permitiendo crear gran número de máquinas virtuales, sin causar problemas de consumo.

Desventajas

-Hay incompatibilidad con otros sistemas que no tengan el Kernel, para su funcionamiento en otros sistemas es necesario tener Kernel y arreglar la conexión al Docker
-Los sistemas operativos virtuales que se crean son Linux.

Recursos

Utiliza recursos a nivel de kernel y dos herramientas de aislamiento general(cgroup y namespaces)
Quiere decir que los ficheros de la máquina virtual pueden ser accedidos desde el host físico, pero la máquina virtual no puede acceder a nada fuera de ella.
Permiten aislar el cpu y memoria que se va a usar, personalizandolo hasta ciertos límites

Utilidad

Docker consigue ser flexible y bastante portable, permitiendo migrar estos sistemas operativos de una máquina a otra, copiar, extender etc.
Se puede crear un registro en cloud de un Docker que permitirá usarlo desde cualquier lugar con acceso al internet.
Esta enfocado a sistemas distribuidos, creando copias de entorno TI
